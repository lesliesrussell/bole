## What it is

Access-controlled, content-addressed version control **library** in Rust (~32k LOC across 145
files), plus two thin frontends. Core thesis: put access control *inside* the repo вЂ” actors,
label lattices, ACLs, and signed policy are first-class content-addressed objects in the same
BLAKE3 store as files and history. Mission (sharpened 2026-07-04): **bole = headless
backend/API for "Grove"**, a planned GitHub-alternative frontend living in a separate future
repo.

Three workspace crates:

| Crate | Role |
|---|---|
| `bole` (src/, 64 files, 18.7k LOC) | The library: everything below |
| `bole-cli` (53 files, 8.1k LOC) | Thin CLI wrapper, 25 command modules |
| `bole-api` (16 files, 1.4k LOC) | axum read-only HTTP/JSON server (shipped 2026-07-04/05) |

## How it works вЂ” subsystem map

- **`object/`** вЂ” typed content-addressed objects: `Blob`, `Tree`, `Snapshot` (immutable DAG
  node), `Secret`/`SecretV2`/`MultiRecipientSecret` (envelope-encrypted), `EnvOverlay`. All
  keyed by 32-byte BLAKE3 `ObjectId`.
- **`store/`** вЂ” `ObjectStore` faГ§ade over pluggable `StorageBackend` (memory + packed disk).
  Pack format with mark-sweep GC; hard resource caps (128 MiB object, 1 GiB pack, etc.).
- **`refs/`** вЂ” mutable pointers: `Timeline` (movable, advancement policy
  `ff`/`append`/`unrestricted`) and `Tag` (fixed). Atomic multi-ref transactions.
- **`acl/`** вЂ” largest, highest-complexity module. Label lattice with scoped clearances (glob
  ACLs are the degenerate two-point case), `Accessor` credentials, memoized glob matcher
  (ReDoS-hardened), content-addressed `PolicyHook`s (`signed-approval` gates advance/merge via
  Ed25519 attestations against an approver registry), detached-signature policy authority chain
  (`verify_chain`/`reconcile`, fail-closed).
- **`repo/`** вЂ” `Repository` orchestration hub (`mod.rs` 2,183 LOC вЂ” known split candidate).
  ACL-filtered snapshot views (`FilteredSnapshot`), merge machinery, workspaces, git import +
  ACL-filtered git export round-trip.
- **`sync/`** вЂ” negotiated pack transfer with CAS-on-heads, TCP + HTTP transports,
  signed-request authn, bounded decode everywhere. **No TLS, no transport peer auth** вЂ”
  deliberate, documented in the threat model.
- **`collab/` + `sync/collab.rs`** вЂ” discovery/relay stack, deepest-built pillar (10 shipped
  slices, WS8aв†’f-c1): signed `Profile`s, `TrustEdge` graph with trust-path ranking, relay
  pinning, multi-relay query, server-side search with cost bounds.
- **`crypto/`** вЂ” key providers (local + chain), KMS stub. ChaCha20-Poly1305, Ed25519, BLAKE3;
  all Ed25519 payloads domain-separated.

Highest-churn hotspots: `src/lib.rs` (0.61), `src/sync/collab.rs` (0.47), `src/repo/mod.rs`
(0.42) вЂ” churn tracks the recent discovery-stack and API work, not instability.

## Remaining problems

Three tiers, all tracked (25 open beads at audit time, none blocked or in-progress).

### 1. Security gaps a deployer must know

From the threat model "known limitations" + P1/P2 security beads:

- No TLS or transport peer authentication вЂ” must not expose a listener on an untrusted
  network. External crypto audit never done.
- Signatures domain-separated per scheme but **not repo-bound** вЂ” same key trusted in two
  repos allows cross-repo replay. Mitigation today: distinct keys per repo.
- Signed approvals gate **local** operations only; replicated push into approval-gated
  timelines is refused fail-closed rather than enforced remotely. Deterministic "approval
  commit" is future work.
- `bole-au0t` (P1): replicas seeing unknown `PolicyHook` kinds must fail closed вЂ” distributed
  policy soundness, open.
- `bole-e78l` (P1): `scoped/` refs can leak via serve paths + publish TOCTOU on profile/edge
  writes.
- Secret revocation is forward-only (dropped key wrap в‰  rotated data key).
- `bole-api` follow-ups: unknown-bearer-token contract (`bole-261x`), keyId enumeration oracle
  + proxy IP handling (`bole-6lzk`), debug router gating (`bole-gejz`), O(all-refs) timeline
  ACL scan (`bole-tgr8`), non-JSON error paths (`bole-rvyl`). No audit logging of access
  decisions (`bole-eean`).

### 2. The real product gap вЂ” mission-level, not code-level

The 2026-07-04 handoff paused the project deliberately: substrate over-built, product surface
unbuilt. Discoverability got 10 slices of depth; the pillars a user would actually touch вЂ” **PR
system (`bole-j899`), message board (`bole-bz4s`), dev landing page / profile-bundle
(`bole-k93a`, designed + planned but not built)** вЂ” are at zero. The read-only `bole-api`
server shipped after the handoff, partially answering the transport question (`bole-yd56`),
but there are no write endpoints yet (they were gated on the now-fixed `bole-e333`, so that is
unblocked).

### 3. Deliberate deferrals (frozen by scope audit)

Discovery tail (rate limits `bole-tpvl`, denylists `bole-biq1`, querier filters `bole-ldiz`,
reputation `bole-sdny`, DNS alias `bole-o9ia`), git round-trip gaps (`bole-ttik`: annotated
tags, submodules, thin packs), node liveness daemon (`bole-z5rr`), persistent search index
(`bole-lbd1`). One design decision rotting: `MultiRecipientSecret` is built and persistable
but has **no consuming feature** (`bole-oea4`) вЂ” wire it or delete it.

## Bottom line

Codebase is unusually healthy вЂ” no cycles, no dead weight, no rot markers, disciplined
bead-tagged provenance on every block. The remaining problems are not bugs; they are (a) a
short list of tracked security hardening items (two P1s: unknown-hook fail-closed, scoped-ref
serve leak), and (b) the open strategic decision the handoff left on the table: stop deepening
the substrate and build the Grove-facing API surface (profile-bundle в†’ PR system). The
profile-bundle work is fully specced and planned (`bole-k93a`) вЂ” the natural next move.
